# IMPROVEMENT-WORKSPACE-V1

**Date:** 2026-05-22
**Status:** Draft for review (sub-spec #2 of `KERNOS-AUTONOMOUS-IMPROVEMENT-LOOP-V1`)
**Scope:** Worktree lifecycle (create / use / remove) + branch
  contract + workspace-guard primitive + retention policy.
  Substrate-owned: the worktree operations themselves are
  subprocess-level, NOT exposed as agent-callable tools.
  Required dependency for `GIT-OPERATIONS-PRIMITIVES-V1`.
**Estimated size:** ~200 LOC source + ~150 LOC tests.

## Why this spec exists

Per `KERNOS-AUTONOMOUS-IMPROVEMENT-LOOP-V1` D1: the
autonomous-improvement loop needs an isolated working tree
where the coding agent can edit Kernos's source without
disturbing the live process. A git worktree at
`data/<instance>/improvement_workspace/<attempt_id>/` provides
that isolation for v1.

**Trust boundary (load-bearing):** the worktree is NOT a
security sandbox. The coding agent invoked via `consult`
inherits credentials + filesystem write outside the worktree
+ ability to push to origin. v1 ships against TRUSTED CODING
AGENTS only (`claude_code`, `codex` — operator-vetted CLIs).
v2 closes the trust gap by replacing the worktree with a
container-backed primitive without changing the orchestration
workflow.

This sub-spec ships the substrate that owns workspace lifecycle.
Git operations on an already-created worktree
(`git_fetch`, `git_commit`, etc.) are agent-callable kernel
tools shipped in `GIT-OPERATIONS-PRIMITIVES-V1` — they consume
the workspace guard from this spec.

## Design principles (load-bearing)

Per [[agent-facing-natural-simplicity]]: this sub-spec is
substrate-internal. The agent does NOT call worktree
create/remove directly — the orchestrator workflow
(`IMPROVEMENT-LOOP-WORKFLOW-V1`) calls them. Agent's experience
of the workspace is via the git-ops kernel tools (next sub-spec),
which surface errors as natural prose via the guard's response
shape.

The workspace guard's REJECTION message IS agent-facing prose
(consumed by git tools when a `workspace_dir` argument doesn't
validate). Substrate keeps the structured reason in logs +
diagnostic events; agent gets a sentence.

## Current state

- No improvement workspace exists today. `data/<instance>/` has
  spaces, conversation logs, friction reports, etc. — nothing
  for improvement attempts.
- Git operations today are confined to the LIVE working tree
  (the live process's `kernos/` clone). No worktree primitive.
- `data_dir` env (`KERNOS_DATA_DIR`) tells the substrate
  where instance state lives; this spec adds an
  `improvement_workspace/` subdirectory underneath.

## Design

### Directory layout

```
data/<instance>/improvement_workspace/
  <attempt_id>/             # one git worktree per attempt
    .git                    # worktree's git metadata (points back to live repo)
    kernos/                 # the Kernos source tree (full mirror)
    ...
```

`<attempt_id>` is a UUID hex generated by the orchestrator at
attempt start (single-source: shared with the ledger).

### Worktree lifecycle

```python
class ImprovementWorkspace:
    def __init__(self, data_dir: str, instance_id: str,
                 live_repo_dir: str) -> None:
        """live_repo_dir = path to the running Kernos's git repo
        (e.g., ~/Kernos). Worktrees branch off this repo's
        origin/main."""
        ...

    async def create(self, attempt_id: str) -> str:
        """Create a worktree at data/<instance>/improvement_workspace/<attempt_id>/
        on branch improvement/<attempt_id> from origin/main.
        Returns the absolute worktree path.

        Side-effects:
          - shells out: `git fetch origin` (in live repo)
          - shells out: `git worktree add -b improvement/<attempt_id>
                         <worktree_path> origin/main`

        Raises WorkspaceCreateError on failure.
        """
        ...

    async def remove(self, attempt_id: str, *,
                     force: bool = False) -> None:
        """Remove the worktree (and delete its branch unless the
        branch has un-pushed commits AND force=False, in which case
        raises WorkspaceRemoveError for operator review).

        Side-effects:
          - shells out: `git worktree remove <worktree_path>` (or
                         `--force` when force=True)
          - shells out: `git branch -D improvement/<attempt_id>` (best-effort)
        """
        ...

    def path_for(self, attempt_id: str) -> str:
        """Return the absolute path where this attempt's worktree
        lives. Pure path computation; doesn't create the directory."""
        ...

    async def list_active(self) -> list[dict]:
        """Return list of active worktrees: {attempt_id, path,
        branch, created_at, age_days}. Reads from `git worktree list`."""
        ...

    async def cleanup_expired(self, *, max_age_days: int = 7) -> int:
        """Remove worktrees older than max_age_days. Returns count
        removed. Called by a background task (the orchestrator's
        retention loop)."""
        ...
```

### Workspace guard

The guard is the load-bearing safety primitive consumed by
git-ops tools. Validates that any `workspace_dir` argument is
under the instance's `improvement_workspace/` root.

```python
def validate_workspace_path(
    *, claimed_path: str, instance_id: str, data_dir: str,
) -> tuple[bool, str]:
    """Returns (ok, reason). ok=True when claimed_path is a real,
    existing directory under
    data/<instance>/improvement_workspace/<attempt_id>/.

    Defends against:
      - path traversal (".." in segments)
      - symlink escape (resolves to absolute path, checks parent)
      - typoed instance_id (must match the caller's instance)
      - claimed_path pointing to a non-worktree directory
      - claimed_path pointing OUTSIDE improvement_workspace/

    reason is operator-facing on failure (logs); ALSO doubles
    as the agent-facing message via the git-ops tools' error
    output composer (those tools call into agent-prose composition
    using this reason).
    """
    ...
```

### Branch contract (per parent spec D1.1)

- Worktree branch: `improvement/<attempt_id>` (NOT `main`).
- Branch created from `origin/main` at attempt start.
- Worktree branch is local-only until orchestrator's Step 6 push
  (which uses `git_push` with expected-parent-SHA enforcement).
- Branch deleted on worktree removal (unless un-pushed commits
  exist AND force=False).

### Retention policy

Default: 7 days. Configurable via
`KERNOS_IMPROVEMENT_WORKSPACE_RETENTION_DAYS` env. The
background retention loop runs in the orchestrator workflow
(not this sub-spec); this spec ships the
`cleanup_expired(max_age_days)` method it calls.

### Trust-boundary documentation surface

The class docstring + `kernos/kernel/improvement_workspace.py`
module docstring carry the trust-boundary statement verbatim
from the parent spec D1. Every PR that touches this module
sees it. Future-Kernos must not silently demote the warning to
a comment that drifts.

### What this sub-spec does NOT ship

- Agent-callable worktree create/remove — substrate-owned only.
- Git operations on the worktree (fetch, commit, push, etc.) —
  ship in `GIT-OPERATIONS-PRIMITIVES-V1`.
- The orchestrator workflow that calls `create` / `remove` —
  ships in `IMPROVEMENT-LOOP-WORKFLOW-V1`.
- Container/sandbox upgrade — v2 work.
- Retention background loop — runs in orchestrator workflow.

## Acceptance criteria

| AC | Description |
|---|---|
| AC1 | `ImprovementWorkspace.create(attempt_id)` creates a worktree at the documented path on branch `improvement/<attempt_id>`. |
| AC2 | `create()` fetches origin first so the worktree is up to date. |
| AC3 | `create()` raises `WorkspaceCreateError` when git operations fail (e.g., branch name collision, disk full). |
| AC4 | `path_for(attempt_id)` returns a deterministic absolute path under the instance's `improvement_workspace/`. |
| AC5 | `remove(attempt_id)` removes the worktree + deletes its branch when the branch has no un-pushed commits. |
| AC6 | `remove(attempt_id, force=False)` raises `WorkspaceRemoveError` when the branch has un-pushed commits. |
| AC7 | `remove(attempt_id, force=True)` removes regardless of un-pushed commits. |
| AC8 | `list_active()` returns a list of dicts with `attempt_id, path, branch, created_at, age_days` for every active worktree. |
| AC9 | `cleanup_expired(max_age_days=N)` removes worktrees older than N days; returns the count removed. |
| AC10 | `validate_workspace_path(claimed_path=X, instance_id=I, data_dir=D)` returns `(True, "")` for a real directory under `D/<I>/improvement_workspace/<attempt_id>/`. |
| AC11 | `validate_workspace_path` rejects path traversal (`..` in segments) with `(False, <reason>)`. |
| AC12 | `validate_workspace_path` rejects symlink-escape (path resolves outside improvement_workspace). |
| AC13 | `validate_workspace_path` rejects typoed instance_id (different instance's workspace). |
| AC14 | `validate_workspace_path` rejects non-existent claimed_path. |
| AC15 | Trust-boundary statement present verbatim in module + class docstrings (parity test asserts the canonical phrase). |
| AC16 | No regressions — no test changes elsewhere expected since this is purely additive substrate. |

## Soak gate

1. **Automated**: ACs pin lifecycle + guard + retention + trust-
   boundary docstring presence.
2. **Operator soak**:
   - Run `ImprovementWorkspace.create("test-abc")` against
     a real Kernos repo.
   - Verify the worktree exists at the expected path.
   - Verify branch `improvement/test-abc` exists.
   - Modify a file in the worktree (operator-side test);
     verify it doesn't affect the live working tree.
   - `remove("test-abc")` — verify worktree + branch are gone.

## Out of scope

- Agent-callable worktree management — substrate-owned per design.
- Git operations on the worktree → `GIT-OPERATIONS-PRIMITIVES-V1`.
- Container sandbox → v2.
- Retention background loop → orchestrator workflow.

## Risks

- **Risk:** The guard fails open on a subtle path-handling bug
  and a git tool writes outside the workspace.
  - **Mitigation:** Test all bypass attempts (path traversal,
    symlink, encoded segments). The guard returns
    `(False, reason)` on any uncertainty rather than passing
    through. Failed-guard tests pin the negative behavior.

- **Risk:** Orphan worktrees accumulate when an attempt
  crashes between `create` and orchestrator's cleanup.
  - **Mitigation:** `cleanup_expired` covers the long-tail
    case. Operator can also manually wipe
    `data/<instance>/improvement_workspace/` if needed.

- **Risk:** The trusted-agent model is documented but operators
  could miss the boundary and run untrusted coding agents.
  - **Mitigation:** Docstring + spec text + `improve_kernos`
    will only dispatch to whitelisted targets (`claude_code`,
    `codex`). Adding a target to the whitelist requires an
    explicit code change → reviewable.

## Dependencies

- `git` binary available on the runtime host (already required).
- `KERNOS_DATA_DIR` env (existing).
- `live_repo_dir` discoverable at runtime — passed in by the
  orchestrator (which knows the Kernos process's own clone path
  via the running binary's resolve).

## Migration

- **Schema**: no DB schema. Filesystem-only.
- **Existing data**: nothing to migrate.
- **Disk usage**: each worktree is a full mirror of Kernos. With
  retention 7d and ~5 attempts/day, expect ~35 mirrors at any
  time. Each ~50MB of source = ~1.75GB worst case. Documented;
  retention is configurable.
